Setup Guidev2.1 Pilot-Safe Build

GWM Daily Sales Reporting, v2.1 setup

Dealer input page to Google Sheets to Power BI. No internal dashboard layer.

1. What changed from v2

  • Forecast has been removed from the daily grid to avoid accidental forecast zeroing.
  • The page now waits for a backend receipt before showing success.
  • Apps Script validates dealer, report date, direction, model buckets and numbers server-side.
  • Active rows go to submissions_current.
  • Every submission version is retained in submissions_audit.
  • Rejected/failed submissions are logged in error_log.
  • Dealer, model and config tabs are created by the setup function.

2. Files to publish

  1. Upload index.html, styles.css, gwm-wordmark.png and user-guide.html to GitHub Pages or the chosen static host.
  2. Replace the old live /user-guide.html. Do not leave the dashboard-era guide online.
  3. Do not publish dashboard.html. It is not part of this build.
  4. Keep setup-guide.html for internal handover.

3. Google Sheet setup

  1. Create or open the testing Google Sheet.
  2. Open Extensions > Apps Script.
  3. Paste the full contents of code.gs.
  4. Replace YOUR_GOOGLE_SHEET_ID_HERE with the actual Google Sheet ID, unless the script is bound to the Sheet.
  5. Save the project as GWM Daily Reporting v2.1.
  6. Run initWorkbookForPilot once and authorise the script.
  7. Run smokeTestSetup. It should confirm 19 active models and a sample dealer.

The setup function creates submissions_current, submissions_audit, dealers, model_buckets, config, error_log, forecast_current and forecast_audit.

4. Security setup for pilot

Apps Script Web App access may still need to be set to Anyone for the fast pilot. That means endpoint protection must come from validation, optional access code and dealer tokens.

  1. In the config tab, optionally set pilot_access_code. Dealers enter this on the form.
  2. For stronger control, add a unique dealer_token in the dealers tab for each dealer.
  3. Dealer-specific links can then use ?dealer=H3100&token=DEALER_TOKEN.
  4. Keep the Google Sheet private to the minimum required GWM users.
  5. For CSV access, optionally set read_csv_token in config.

5. Deploy Apps Script Web App

  1. Go to Deploy > New deployment.
  2. Choose Web app.
  3. Set Execute as to Me.
  4. Set Who has access to Anyone for pilot use.
  5. Deploy and copy the Web App URL.
  6. Open index.html and replace PASTE_DEPLOYED_APPS_SCRIPT_WEB_APP_URL_HERE with the deployed URL.
  7. Republish index.html.

6. Power BI connection

Preferred Apps Script CSV endpoints:

WEB_APP_URL?format=csv&sheet=current
WEB_APP_URL?format=csv&sheet=dealers
WEB_APP_URL?format=csv&sheet=models
WEB_APP_URL?format=csv&sheet=forecast_current

If a read token is configured:

WEB_APP_URL?format=csv&sheet=current&token=YOUR_READ_TOKEN

7. Internal smoke test

  1. Open the live index.html.
  2. Select one dealer, yesterday’s date and a direction.
  3. Enter small test quantities for two model buckets.
  4. Submit and confirm the page shows a receipt.
  5. Confirm exactly 19 rows appear in submissions_current.
  6. Confirm 19 rows also appear in submissions_audit.
  7. Submit the same dealer/date again and confirm submissions_current still has 19 active rows while submissions_audit has another 19 rows.
  8. Try an all-zero submission and confirm the warning appears.
  9. Try today’s date and confirm it is blocked by the page and backend.
  10. Refresh Power BI and confirm rows appear with correct data types.

8. Reset and archive

The ZIP does not reset the live Google Sheet. Reset must be run manually in Apps Script.

  • archiveCurrentBeforeReset copies submissions_current before reset.
  • resetCurrentForTesting clears only submissions_current. Audit remains.
  • resetAllPilotData clears current, audit, errors and forecast tabs. Use only before internal testing, never mid-rollout.

9. Rollback

  • GitHub Pages: revert the previous commit or republish the previous ZIP.
  • Apps Script: Deploy > Manage deployments > edit deployment > select previous version.
  • Power BI: switch source back to the previous CSV endpoint if required.
  • Data: restore from submissions_audit or an archive tab.

10. Go/no-go checklist

  • Old live /user-guide.html replaced.
  • APPS_SCRIPT_URL pasted into index.html.
  • initWorkbookForPilot and smokeTestSetup passed.
  • Receipt displayed after successful submit.
  • All-zero warning tested.
  • Server rejects today/future dates.
  • 19 active rows written per dealer/date.
  • Audit rows retained after correction.
  • Power BI refresh works from current, dealers and model tables.
  • Named business owner, technical owner and fallback owner confirmed.
